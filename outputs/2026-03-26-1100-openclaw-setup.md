## Decision confirmed

OpenClaw setup lives in `idea`. `app-openclaw` is a generic platform reference. The App Disk model is right for school apps (Kolibri, Nextcloud) — not for the IDEA development environment.

## Files created — idea PR #7 (`feat/idea-openclaw-setup`)

### `idea/openclaw/openclaw.json`
JSON5 template — full IDEA agent config derived from the running system:
- 5 agents: Axle, Pixel, Beacon, Marco, Atlas (with real agent IDs and workspace paths)
- Telegram bindings (all 5 group IDs)
- Model defaults (claude-sonnet-4-6, cache retention, compaction)
- Channel config with `{{TELEGRAM_BOT_TOKEN}}` placeholder
- `gateway.remote.token` intentionally absent (auto-generated by docker-setup.sh, merged by setup.sh)

### `idea/openclaw/.env.template`
Credential placeholders: ANTHROPIC_API_KEY, TELEGRAM_BOT_TOKEN, GITHUB_TOKEN, AUTH_TOKEN (MC).
`.env` is gitignored.

### `idea/openclaw/README.md`
How to use the folder, update the roster, regenerate credentials.

### `idea/scripts/setup.sh`
Idempotent install script — 8 steps:
1. System dependencies (Docker, Compose plugin, git, tmux, Node 22, python3)
2. Clone 5 agent repos to `agents/`
3. Prompt for credentials → write `openclaw/.env`
4. Copy `.env` to each agent workspace
5. Clone openclaw repo, run `docker-setup.sh` with `OPENCLAW_EXTRA_MOUNTS=/home/pi/idea:/home/node/workspace:rw`, then apply IDEA openclaw.json (Python merges configs, preserves auto-generated gateway token)
6. Tailscale install + auth
7. Summary with URLs and next steps
